## 1. Architecture Decision: Why Hierarchical Crop-and-Detect

Every team that solves small object detection in production converges on the same architecture: **don't detect small objects in full frames.** Full-frame detection on 1080p with 30-50px logos is a losing game — even SOTA detectors plateau at ~40% mAP for objects <32px.

### The Pipeline (concrete, no options)

```
┌─────────────────────────────────────────────────────────────────┐
│                    FULL PIPELINE — 3 STAGES                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │ STAGE 1: Person Detection + Multi-Object Tracking        │   │
│  │                                                          │   │
│  │ Input:  Full frame (1920×1080)                           │   │
│  │ Model:  YOLOv11l (pretrained COCO, class=person)         │   │
│  │ Tracker: BoTSORT (appearance + motion)                   │   │
│  │ Output: Tracked player bboxes with stable IDs            │   │
│  │         {track_id: bbox} per frame                       │   │
│  └──────────────┬───────────────────────────────────────────┘   │
│                 │                                               │
│                 ▼                                               │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │ STAGE 2: Crop-and-Detect (Logo Detection)                │   │
│  │                                                          │   │
│  │ Input:  Player crops (padded 20%), resized to 640×640    │   │
│  │ Model:  YOLOv11m fine-tuned on YOUR 21 logo classes      │   │
│  │ Output: Per-crop logo detections with class + confidence │   │
│  │                                                          │   │
│  │ KEY: Logo that was 40px in full frame → 150-250px        │   │
│  │      in player crop. This is a 4-6x magnification       │   │
│  │      that transforms the problem from "hard" to "easy".  │   │
│  └──────────────┬───────────────────────────────────────────┘   │
│                 │                                               │
│                 ▼                                               │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │ STAGE 3: Temporal Aggregation + Exposure Measurement     │   │
│  │                                                          │   │
│  │ Per-track logo binding (majority vote across keyframes)  │   │
│  │ Temporal smoothing (fill gaps in detection)              │   │
│  │ Quality-weighted exposure calculation                    │   │
│  │ Output: Per-logo exposure time, QI, segments             │   │

```

## 2. Stage 1: Person Detection + Tracking

### Model: YOLOv11l (pretrained)

**Why YOLOv11l, not YOLOv8 or RT-DETR:**
- YOLOv11l is the latest Ultralytics release. On COCO person class, it achieves ~54.4 mAP@50-95, beating v8l by ~1.5 points
- Person detection is a solved problem — pretrained weights are sufficient, no fine-tuning needed
- The `l` variant (large) is the sweet spot: `m` misses distant/occluded players, `x` is overkill for person detection, and `l` fits comfortably on T4 GPU at 30fps inference

**Why NOT RT-DETR:**
- RT-DETR has better mAP on COCO overall, but for single-class person detection, YOLO's anchor-based approach is more robust for overlapping players (scrum, maul, tackle). Transformer-based detectors handle overlapping objects worse in my experience

```python
from ultralytics import YOLO

# Stage 1: Person detection
person_model = YOLO("yolo11l.pt")  # pretrained COCO

results = person_model.predict(
    frame,
    classes=[0],          # person only
    conf=0.35,            # lower threshold — we'd rather have FP persons than miss
    iou=0.5,              # moderate NMS — rugby players overlap a lot
    imgsz=1280,           # upscale for better small person detection
    half=True,            # fp16 on GPU
    verbose=False,
)
```

### Tracker: BoTSORT (not ByteTrack)

**Why BoTSORT over ByteTrack:**
- ByteTrack is motion-only (Kalman filter). When a player is occluded behind another for 1-2 seconds (tackle, ruck), ByteTrack loses the track and assigns a new ID
- BoTSORT adds **appearance features** (ReID) — it can re-identify the same player after occlusion by jersey color/number/appearance
- For exposure measurement, stable track IDs are **critical** — a split track means split exposure data

```python
# Using supervision library for clean tracking API
import supervision as sv

tracker = sv.ByteTrack(  # supervision wraps BoTSORT too
    track_activation_threshold=0.35,
    lost_track_buffer=90,        # keep track alive for 3 seconds at 30fps
    minimum_matching_threshold=0.8,
    frame_rate=30,
)

# Per frame
detections = sv.Detections.from_ultralytics(results[0])
tracked = tracker.update_with_detections(detections)
# tracked.tracker_id → stable IDs
```

**Configuration tuning for rugby specifically:**
```python

MATCH_THRESHOLD = 0.7   # slightly lower to handle jersey similarity
```

## 3. Stage 2: Crop-and-Detect (THE CORE)

### Why Crop-and-Detect is non-negotiable

This is the single most impactful architectural decision. Numbers:

| Detection target | Full frame (1920×1080) | Player crop (640×640) |
|-----------------|------------------------|----------------------|
| Logo pixel size | 30-50px | 150-250px |
| mAP@50 (estimated) | 35-45% | 75-85% |
| Background clutter | Field, crowd, ads, LED | Just jersey + limbs |
| Compute per frame | 1× inference | N × inference (N=players) |

The compute increase is worth it: typical rugby frame has 6-10 visible Bradford players → 6-10 additional small-image inferences, which is still faster than one inference on a large image with higher resolution.

### Crop Extraction

```python
def extract_player_crops(frame, tracked_detections, pad_ratio=0.2):
    """Extract padded player crops for logo detection."""
    crops = []
    h, w = frame.shape[:2]
    
    for det in tracked_detections:
        x1, y1, x2, y2 = det.bbox
        bw, bh = x2 - x1, y2 - y1
        
        # Minimum size filter — too small = too far from camera
        if bh < 80:  # Less than 80px tall — logo is <15px, undetectable
            continue
        
        # Pad 20% for context (jersey extends slightly beyond bbox)
        px, py = int(bw * pad_ratio), int(bh * pad_ratio)
        cx1 = max(0, x1 - px)
        cy1 = max(0, y1 - py)
        cx2 = min(w, x2 + px)
        cy2 = min(h, y2 + py)
        
        crop = frame[cy1:cy2, cx1:cx2]
        
        crops.append({
            "crop": crop,
            "track_id": det.tracker_id,
            "original_bbox": (x1, y1, x2, y2),
            "crop_bbox": (cx1, cy1, cx2, cy2),
            "player_height_px": bh,
        })
    
    return crops
```

### Logo Detection Model: YOLOv11m fine-tuned

**Why YOLOv11m (medium), not l or s:**
- Input is 640×640 crops (not 1920×1080) → smaller model is sufficient
- `m` variant: 20.1M params, 68.0 GFLOPs — fast enough for batch inference on crops
- `l` would be marginal improvement for 2x compute cost on crops
- `s` would miss subtle logos (partially occluded, low contrast)

**Why YOLO over DINO-based detectors for this stage:**
- You have exactly 21 known, fixed classes — not open-vocabulary
- YOLO's confident box regression outperforms transformer queries for precise small bbox localization on known classes
- Training is simpler: Roboflow → YOLO export → fine-tune in 2 hours on Colab

### Training Data Preparation (CRITICAL)

> [!IMPORTANT]
> **How you prepare training data determines 80% of your final accuracy.** The model architecture matters far less.

#### Annotation strategy on Roboflow

```
1. Annotate on FULL FRAMES (not crops)
   - Annotator sees full context → fewer labeling errors
   - Roboflow stores bbox in absolute coordinates
   
2. During TRAINING, auto-crop:
   - For each annotated frame, run person detection
   - Extract player crops
   - Map logo annotations into crop coordinate space
   - Train YOLO on CROPS, not full frames

3. This means:
   - 400 annotated frames × ~8 players/frame = ~3200 training crops
   - Each crop has 1-4 logo annotations
   - Much richer effective dataset than 400 full-frame images
```

#### Implementation: Full-frame annotation → Crop-based training

```python
def convert_fullframe_to_crops(annotation_data, frame, person_detections):
    """Convert full-frame annotations to per-player-crop annotations."""
    crop_annotations = []
    
    for person in person_detections:
        px1, py1, px2, py2 = person.bbox_padded
        crop = frame[py1:py2, px1:px2]
        crop_h, crop_w = crop.shape[:2]
        
        # Find which logo annotations fall INSIDE this person crop
        crop_logos = []
        for logo in annotation_data.logos:
            lx1, ly1, lx2, ly2 = logo.bbox
            # Check overlap with person crop
            overlap_x1 = max(lx1, px1)
            overlap_y1 = max(ly1, py1)
            overlap_x2 = min(lx2, px2)
            overlap_y2 = min(ly2, py2)
            
            if overlap_x2 > overlap_x1 and overlap_y2 > overlap_y1:
                # Remap to crop coordinates
                new_bbox = (
                    (overlap_x1 - px1) / crop_w,
                    (overlap_y1 - py1) / crop_h,
                    (overlap_x2 - px1) / crop_w,
                    (overlap_y2 - py1) / crop_h,
                )
                crop_logos.append({
                    "class": logo.class_name,
                    "bbox_norm": new_bbox,
                })
        
        crop_annotations.append({
            "image": crop,
            "annotations": crop_logos,
        })
    
    return crop_annotations
```

## 4. Stage 3: Temporal Aggregation + Exposure Engine

### 4.1 Team Classification

Before logo inference, identify which tracked players belong to Bradford:

```python
def classify_team_color(crop_bgr, team_config):
    """
    Classify team by dominant jersey color in torso region.
    Simple and robust — no ML needed.
    """
    h, w = crop_bgr.shape[:2]
    # Torso region: 20%-60% of player height (skip head, skip legs)
    torso = crop_bgr[int(h*0.2):int(h*0.6), int(w*0.2):int(w*0.8)]
    
    # Convert to HSV for color matching
    hsv = cv2.cvtColor(torso, cv2.COLOR_BGR2HSV)
    
    # For Bradford Black kit:
    # Black: V < 60 (dark) and S < 80 (not saturated)
    black_mask = (hsv[:,:,2] < 60) & (hsv[:,:,1] < 80)
    black_ratio = black_mask.mean()
    
    # For Bradford White kit:
    # White: V > 180 (bright) and S < 50 (desaturated)
    white_mask = (hsv[:,:,2] > 180) & (hsv[:,:,1] < 50)
    white_ratio = white_mask.mean()
    
    if black_ratio > 0.35:
        return "bradford_home"  # Black kit
    elif white_ratio > 0.35:
        return "bradford_away"  # White kit
    else:
        return "opponent"
```

> [!TIP]
> Color histogram is sufficient here. Don't over-engineer with a CNN classifier — jersey colors are distinct enough that simple thresholding works. Save ML complexity for where it matters (logo detection).

### 4.2 Logo-Player Binding

```python
class LogoTracker:
    """Accumulate logo detections per tracked player."""
    
    def __init__(self, min_binding_confidence=0.6, min_detection_count=3):
        self.tracks = {}  # {track_id: TrackData}
        self.min_conf = min_binding_confidence
        self.min_count = min_detection_count
    
    def update(self, track_id, logo_detections, frame_num):
        if track_id not in self.tracks:
            self.tracks[track_id] = TrackData(track_id)
        
        track = self.tracks[track_id]
        track.frames_seen += 1
        
        for det in logo_detections:
            if det.confidence >= self.min_conf:
                track.logo_counts[det.class_name] += 1
                track.logo_confidences[det.class_name].append(det.confidence)
    
    def get_bound_logos(self, track_id):
        """Return confirmed logos for this player track."""
        track = self.tracks.get(track_id)
        if not track:
            return []
        
        bound = []
        for logo_class, count in track.logo_counts.items():
            if count >= self.min_count:
                avg_conf = np.mean(track.logo_confidences[logo_class])
                bound.append({
                    "class": logo_class,
                    "binding_confidence": avg_conf,
                    "detection_count": count,
                    "detection_rate": count / track.frames_seen,
                })
        return bound
```

### 4.3 Selective Keyframe Detection

You don't need to run logo detection on every frame — that's wasteful:

```python
def select_keyframes(tracked_players, fps=30, strategy="adaptive"):
    """
    Select which frames to run expensive logo detection on.
    
    Strategy: detect logos at 2-3fps for established tracks,
    every frame for new tracks (first 1s to establish binding quickly).
    """
    keyframes = set()
    
    for frame_num, players in tracked_players.items():
        for track_id in players:
            track = logo_tracker.tracks.get(track_id)
            
            if track is None or track.frames_seen < fps:
                # NEW track — detect every frame for first second
                keyframes.add(frame_num)
            elif not track.has_stable_binding():
                # Unstable binding — detect every 5 frames
                if frame_num % 5 == 0:
                    keyframes.add(frame_num)
            else:
                # Stable binding — detect every 15 frames (2fps)
                # Just to CONFIRM binding, not establish it
                if frame_num % 15 == 0:
                    keyframes.add(frame_num)
    
    return keyframes

# Result: instead of 162,000 × N_players logo inferences,
# you run ~20,000 × N_players — 8x reduction in compute
```

### 4.4 Temporal Smoothing + Inference

```python
def compute_exposure(logo_tracker, tracked_players, total_frames, fps):
    """
    Final exposure computation with temporal inference.
    
    Rule: If player is tracked (visible) and has bound logos,
    those logos are considered "exposed" — even if logo detection
    didn't fire on that specific frame.
    """
    exposure = defaultdict(lambda: {
        "raw_frames": 0,
        "detected_frames": 0,
        "inferred_frames": 0,
        "weighted_exposure": 0.0,
        "segments": [],
    })
    
    for frame_num in range(total_frames):
        players = tracked_players.get(frame_num, {})
        
        for track_id, player_data in players.items():
            bound_logos = logo_tracker.get_bound_logos(track_id)
            
            if not bound_logos:
                continue
            
            # Compute visibility weight for this player in this frame
            weight = compute_visibility_weight(
                player_data.bbox,
                frame_w=1920, frame_h=1080,
                n_players_in_frame=len(players),
            )
            
            for logo in bound_logos:
                logo_class = logo["class"]
                exp = exposure[logo_class]
                exp["raw_frames"] += 1
                
                # Was logo actually detected on this frame?
                if frame_num in logo_tracker.detection_frames.get(
                        (track_id, logo_class), set()):
                    exp["detected_frames"] += 1
                    exp["weighted_exposure"] += weight * logo["binding_confidence"] / fps
                else:
                    # Inferred from tracking
                    exp["inferred_frames"] += 1
                    # Lower weight for inferred (0.7× detected weight)
                    exp["weighted_exposure"] += weight * logo["binding_confidence"] * 0.7 / fps
    
    return exposure


def compute_visibility_weight(bbox, frame_w, frame_h, n_players_in_frame):
    """Quality Index for this exposure instance."""
    x1, y1, x2, y2 = bbox
    bw, bh = x2 - x1, y2 - y1
    
    # Size factor: larger player → logo more visible
    size = (bw * bh) / (frame_w * frame_h)
    size_score = min(size * 50, 1.0)  # Cap at 1.0
    
    # Position factor: center of screen → more viewer attention
    cx = (x1 + x2) / 2 / frame_w
    cy = (y1 + y2) / 2 / frame_h
    dist = ((cx - 0.5)**2 + (cy - 0.5)**2) ** 0.5
    position_score = max(0, 1.0 - dist * 1.5)
    
    # Clutter factor: fewer players → less competition for attention
    clutter_score = 1.0 / (1 + 0.1 * n_players_in_frame)
    
    return size_score * position_score * clutter_score
```

## 5. Biggest Failure Cases + Mitigations

### Failure Case 1: Track ID Swap During Ruck/Maul

**Problem**: 6-8 players pile up in a ruck → tracker swaps IDs → logo exposure attributed to wrong player.

**Mitigation**:
```python
# During detected ruck events, freeze logo inference
# Detect ruck: >4 person bboxes with IoU > 0.3 within 200px radius
def is_ruck_event(player_bboxes, iou_threshold=0.3, min_players=4):
    high_overlap_count = 0
    for i, j in combinations(range(len(player_bboxes)), 2):
        if compute_iou(player_bboxes[i], player_bboxes[j]) > iou_threshold:
            high_overlap_count += 1
    return high_overlap_count >= min_players

# During ruck: keep tracking but DON'T update logo bindings
# Resume normal detection after ruck disperses
```

### Failure Case 2: LED Board Logo Confusion

**Problem**: Stadium LED boards display sponsor logos → model detects them as jersey logos.

**Mitigation**:
```python
# LED boards are NEVER inside a tracked person bbox
# Crop-and-detect inherently solves this — you only run logo detection
# on PLAYER CROPS, not the full frame. LED boards are outside crops.
# → This is already handled by the architecture. No additional work needed.
```

### Failure Case 5: Same Logo Different Positions (Front vs Back)

**Problem**: `aon_red` on chest vs `aon_red` on back — both detected as same class but have different exposure profiles.

**Mitigation:**
```python
# Annotate with position-aware classes:
# aon_red_chest, aon_red_back, mcp_home_sleeve_l, mcp_home_sleeve_r
# 
# BUT: this doubles your classes from 21 → 42, and you need MORE training data
# 
# PRAGMATIC: keep 21 classes, but USE BBOX POSITION within crop to infer location:
def infer_logo_position(logo_bbox, player_crop_shape):
    """Infer if logo is front/back/sleeve based on position in player crop."""
    lx_center = (logo_bbox[0] + logo_bbox[2]) / 2
    ly_center = (logo_bbox[1] + logo_bbox[3]) / 2
    ph, pw = player_crop_shape[:2]
    
    # Relative position
    rx, ry = lx_center / pw, ly_center / ph
    
    if 0.3 < rx < 0.7 and 0.2 < ry < 0.5:
        return "chest"  # Center of torso
    elif rx < 0.25 or rx > 0.75:
        return "sleeve"
    elif ry > 0.5:
        return "shorts_or_socks"
    else:
        return "upper"  # Could be chest or back
    
    # Note: front vs back requires pose estimation — out of scope for v1
```

## 6. Evaluation Metrics + Production Validation

### Training Evaluation (on val split)

```python
# Primary metrics:
# mAP@50     — overall detection quality
# mAP@50:95  — stricter, penalizes imprecise boxes
# Per-class AP — identifies weak classes needing more data

# TARGET for crop-based logo detection:
# mAP@50   > 0.75  (good)
# mAP@50   > 0.85  (excellent)
# Per-class AP: no class below 0.50 (if any class < 0.50, add more training data)
```

### End-to-End Validation (CRITICAL)

> [!IMPORTANT]
> Training metrics (mAP) and production accuracy are **different things**. You MUST validate end-to-end.

#### Ground Truth Benchmark Protocol:
```
1. Select 3 video segments: 5 minutes each, from different match phases
   - Segment A: Open play (mixed blur/sharp, varying distances)
   - Segment B: Set piece (lineout/scrum — static, good visibility)  
   - Segment C: Fast play (open running, high motion blur)
   
2. Manual annotation (by YOU, frame by frame):
   - Every frame: which Bradford players visible? Which logos visible?
   - This gives TRUE exposure time per logo for these 15 minutes
   
3. Run your pipeline on same 15 minutes
   
4. Compare:
   - Exposure time deviation: |pipeline_time - true_time| / true_time
```

## 8. Summary — The Non-Negotiables

1. **Crop-and-detect, not full-frame detect.** This is the single biggest accuracy lever.
2. **Motion blur augmentation during training.** Without this, model will fail on 50% of real frames.
3. **Track + Infer, not detect every frame.** Leverages temporal continuity for exposure measurement.
4. **Validate end-to-end with manual ground truth**, not just training mAP.
5. **Precision over recall for logo detection.** False positives destroy credibility with sponsors. Set confidence threshold high (0.5-0.6), compensate with tracking inference.
